# Text-to-SQL Agent ‚Äî Claude Instructions

## Stack

- Next.js 15 (App Router) with TypeScript
- LangGraph + LangChain + Ollama (chat model, zero temperature)
- SQLite3 (local demo database via `src/core/database.ts`)
- Tailwind CSS
- Vitest for testing

## Architecture

- `src/core/actions.ts` ‚Äî server action (`"use server"`) that builds the LangGraph ReAct agent and handles LLM calls
- `src/core/constants.ts` ‚Äî SQL schema definitions and system prompt
- `src/core/database.ts` ‚Äî SQLite query execution
- `src/core/chat-view-model.ts` ‚Äî maps stored messages to UI-friendly shapes
- `src/components/chat-message.tsx` ‚Äî renders individual chat messages
- `src/utils/format-message-content.ts` ‚Äî formats raw LLM content for display

## Build and Test

```bash
npm install          # install dependencies
npm run dev          # start dev server (port 3000)
npm run build        # production build
npm run start        # start on port 3001
npm run test         # run vitest once
npm run test:watch   # vitest watch mode
npm run lint         # ESLint
npm run seed         # seed the SQLite database
```

## Environment Variables

Copy `.env.sample` ‚Üí `.env` and set:

- `OLLAMA_BASE_URL` ‚Äî URL to Ollama server (e.g. `http://localhost:11434`)
- `OLLAMA_MODEL` ‚Äî model name (e.g. `qwen3-coder:30b`)

## Conventions

- All LLM and database calls are server-side only (Next.js `"use server"` action in `src/core/actions.ts`)
- SQL field and table names must always be wrapped in double quotes (`"`) ‚Äî enforced via the system prompt in `src/core/constants.ts`
- Tests live alongside source files (`*.test.ts`); no separate `__tests__` directories
- Do not add new environment variables without updating `.env.sample`

## Guardrails

- Do not generate or execute `DROP`, `TRUNCATE`, or `DELETE` SQL without a scoped `WHERE` clause
- Do not expose `process.env` values or database file paths in responses or client-side code
- Do not move LLM or database logic out of `src/core/actions.ts` into client components
- Do not bypass the `"use server"` boundary ‚Äî all agent calls must remain server-side
- Do not add new npm dependencies without a clear, justified need

## Data Model

Tables in the SQLite database (defined in `src/core/constants.ts`):

**`customer`** ‚Äî `id` (PK), `email`, `name`

**`order`** ‚Äî `id` (PK), `createdate` (TEXT, ISO date), `shippingcost` (REAL), `customerid` (FK ‚Üí customer), `carrier`, `trackingid`

`order.customerid` references `customer.id`. Always quote both table and field names in SQL.

## Testing

- Framework: Vitest with `describe` / `it` / `expect` from `"vitest"`
- No mocking of the database or LLM ‚Äî tests cover pure transformation logic only
- `chat-view-model.test.ts` ‚Äî tests `getRenderableMessages`: filtering, role mapping, deterministic keys
- `format-message-content.test.ts` ‚Äî tests `formatMessageContent`: string passthrough, array joining (text parts only), fallback stringify
- When adding tests, follow the same `describe` ‚Üí `it` structure; keep each test focused on a single behaviour

## Files to Avoid Editing

- `next-env.d.ts` ‚Äî auto-generated by Next.js, do not modify
- `src/app/globals.d.ts` ‚Äî ambient type declarations, edit only for new global types
- `":memory:"` (root) ‚Äî SQLite artifact from the `":memory:"` database path string; do not delete

## Common Pitfalls

- The `":memory:"` file at the project root is a SQLite database file created because the path literal `'":memory:"'` is passed to `sqlite3.Database` ‚Äî it is not an in-memory database
- `zod` is used for tool schema validation inside `actions.ts` but is not listed in `package.json`; it is pulled in transitively via `@langchain/core` ‚Äî do not add it as a direct dependency unless the transitive dependency is removed
- `formatMessageContent` intentionally skips non-text array parts (e.g. images) ‚Äî this is by design, not a bug
